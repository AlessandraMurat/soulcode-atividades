Rotas e navegação com Angular

Mais uma praticidade que o Angular nos proporciona é o esquema de rotas que como outras coisas sao modularizadas, com as rotas não é diferente. Para ter rotas em nossa aplicação basta criarmos um arquivo de rotas que geralmente tem o seguinte nome: app-routing.module.ts e sua estrutura consiste em:

import {{ '{' }} BrowserModule {{ '}' }} from '@angular/platform-browser';

import {{ '{' }} Routes, RouterModule {{ '}' }} from '@angular/router';

const routes: Routes = [];


@NgModule ({{ '{' }}

    imports:[RouterModule.forRoot.(routes)],

    exports:[ RouterModule ]

{{ '}' }})

export class AppRoutingModule {{ '{' }} {{ '}' }}

Compondo as rotas

As rotas são definidas por dois elementos, os quais sao o path e o componente, que é o componente que queremos visualizar.

import {{ '{' }} BrowserModule {{ '}' }} from '@angular/platform-browser';

import {{ '{' }} Routes, RouterModule {{ '}' }} from '@angular/router';

import {{'{'}} ExemploComponent {{ '}' }} from './exemplo/exemplo.component';

const routes: Routes = [{{ '{' }} path:'exemplo', component: ExemploComponent {{ '}' }}];


@NgModule ({{ '{' }}

    imports:[RouterModule.forRoot.(routes)],

    exports:[ RouterModule ]

{{ '}' }})

export class AppRoutingModule {{ '{' }} {{ '}' }}

Navegando via HTML

Ao tentar navegar pelas páginas pelas rotas, o Angular não irá conseguir encontrar onde ele deveria renderizar o componente, para isto é preciso inserir uma tag no arquivo app.component.html, ressaltando que é nesse arquivo que ficará o conteúdo que queremos mostrar em todas as páginas, como exemplo temos o footer e o header..

Para a navegação é primordial esquecer de usar o href padrão do HTML para links de páginas, pois ela irá recarregar a página e voltará para a tela inicial. Ao inv´s disso, iremos substituir pelo routerLink do Angular, que fará com que apenas os componentes sejam redenrizados na tela. Desta forma, você precisa substituir o href pelo código a seguir para navegar nas páginas HTML.

1   |   [routerLink]="/exemplo"
